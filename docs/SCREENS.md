# Экраны и роутинг

## Карта роутов

| Путь | Экран | Описание |
|------|-------|----------|
| `/` | Dashboard | Сводка по всем режимам, точка входа |
| `/study/:modeId` | Study | Активная учебная сессия |
| `/decks` | Decks | Список колод (управление наборами слов) |
| `/decks/:deckId` | Deck Words | Слова внутри конкретной колоды |
| `/stats` | Stats | Графики и статистика прогресса |
| `/settings` | Settings | Настройки приложения |
| `*` | Dashboard | Fallback — редирект на главную |

## Layout

Все экраны обёрнуты в общий `Layout`, который содержит:

- `<Outlet />` — место для рендеринга дочернего роута
- `<NavBar />` — фиксированная нижняя навигация

### NavBar (нижняя навигация)

Фиксированная панель внизу экрана (mobile tab bar pattern). 4 вкладки:

| Вкладка | Иконка | Путь | Ключ i18next |
|---------|--------|------|--------------|
| Главная | домик | `/` | `t('nav.home')` |
| Колоды | коробка | `/decks` | `t('nav.decks')` |
| Статистика | график | `/stats` | `t('nav.stats')` |
| Настройки | шестерёнка | `/settings` | `t('nav.settings')` |

Все подписи берутся из `useTranslation()`. Иконки одинаковы для всех языков.

Активная вкладка выделяется цветом `--color-accent`. Используется `<NavLink>` из `react-router` с callback `className` для определения активного состояния.

**NavBar скрывается** на экране Study (`/study/:modeId`), чтобы не отвлекать во время сессии.

Роутер настраивается в `App.tsx` через `<Routes>` и `<Route>` из `react-router` (декларативный режим, без Framework Mode).

### Назначение
Точка входа. Показывает состояние всех режимов обучения и позволяет начать сессию.

```
│  └────────────────────┘  │
│                          │
│  ┌────────────────────┐  │
│  │ 📚 Словарь         │  │
│  │ 12 слов к повтору  │  │
│  │ [ Учить → ]        │  │
│  └────────────────────┘  │
│                          │
│  ┌────────────────────┐  │
│  │ 🔢 Числа           │  │
│  │ Всегда доступно    │  │
│  │ [ Учить → ]        │  │
│  └────────────────────┘  │
│                          │
│  ┌─────┬─────┬────┬───┐ │

│  └─────┴─────┴────┴───┘ │
└──────────────────────────┘
```

### Данные

- **Общая сводка** (верх):
  - Слов в активных колодах: количество слов из колод с `isActive = true` (через `useLiveQuery`)
  - Слов изучено: количество слов, у которых хотя бы одна CardState в состоянии Review (из активных колод)
  - Средний retention: из `VocabularyMode.getStats().averageRetention`
  - Активных колод: N из M

- **Плитки режимов** (цикл по `getAllModes()`):
  - Название: `t('mode.' + mode.id + '.title')`, иконка из `mode.icon`
  - Количество к повтору: `mode.getDueCount()` (обновляется реактивно)
  - Кнопка `t('dashboard.study')` → навигация к `/study/${mode.id}`

### Действия пользователя

- Нажать "Учить" на плитке → переход к `/study/:modeId`
- Переключение вкладок через NavBar

## Study (/study/:modeId)

### Назначение
Экран учебной сессии. Показывает упражнения одно за другим и подводит итоги.

### Жизненный цикл

```
Loading → In Progress → Finished
                ↓
         (ошибка) → Error
```

### Состояния экрана

#### 1. Loading

Показывается при загрузке карточек. Минимальный спиннер/скелетон.

#### 2. In Progress

```
┌──────────────────────────┐
│  3 / 10          01:23   │  ← прогресс + таймер
│                          │
│  ┌────────────────────┐  │
│  │                    │  │
│  │    casa            │  │  ← вопрос
│  │                    │  │
│  └────────────────────┘  │
│                          │
│  ┌──────┐  ┌──────────┐ │
│  │ дом  │  │ школа    │ │  ← варианты (2x2 grid)
│  └──────┘  └──────────┘ │
│  ┌──────┐  ┌──────────┐ │
│  │ вода │  │ хлеб     │ │
│  └──────┘  └──────────┘ │
│                          │
└──────────────────────────┘
```

Компонент упражнения определяется по `item.exerciseType`:
- `'multiple-choice'` → `<MultipleChoice />`
- `'flip-card'` → `<FlipCard />`
- `'number-input'` → `<NumberInput />`

**Прогресс-бар** вверху: текущий номер / всего.

**Обратная связь после ответа:**
- Правильный вариант подсвечивается зелёным
- Неправильный — красным (+ правильный показывается зелёным)
- Пауза ~1.5 сек, затем автопереход к следующему

#### 3. Finished (SessionSummary)

```
┌──────────────────────────┐
│                          │
│      Сессия завершена    │
│                          │
│      ┌──────────┐        │
│      │   8/10   │        │  ← круговая диаграмма
│      │   80%    │        │
│      └──────────┘        │
│                          │
│    Время: 2 мин 34 сек   │
│                          │
│  ┌────────────────────┐  │
│  │   Учить ещё        │  │
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │   На главную       │  │
│  └────────────────────┘  │
│                          │
└──────────────────────────┘
```

- Правильных / всего с процентом. Определение "правильно" зависит от типа упражнения:
  - Multiple choice: объективно — правильный вариант выбран
  - Flip card: субъективно — пользователь выбрал Good или Easy (рейтинг >= 3)
- Время сессии
- Кнопка "Учить ещё" → начать новую сессию с тем же режимом
- Кнопка "На главную" → `/`

Если `items.length === 0` (нечего повторять) — показать сообщение "Все карточки повторены! Приходите позже."

#### 4. Error

Текст ошибки + кнопка "Попробовать снова" (перезагрузка сессии).

### Хук useSession

Управляет всем жизненным циклом:

```ts
const {
  status,          // 'loading' | 'in-progress' | 'finished' | 'error'
  currentItem,     // текущий SessionItem | null
  currentIndex,    // номер текущего элемента (0-based)
  totalCount,      // всего элементов
  correctCount,    // правильных ответов
  error,           // текст ошибки | null
  startSession,    // () => Promise<void>
  submitAnswer,    // (answer: Answer) => Promise<void>
} = useSession(mode, sessionSize)
```

При завершении сессии хук автоматически сохраняет запись в таблицу `sessions`.

## Decks (/decks)

### Назначение
Управление колодами слов — аналог колод в Anki. Колода объединяет набор слов для изучения. Колоды можно активировать/деактивировать, чтобы управлять тем, какие слова попадают в учебные сессии.

### Структура экрана

```
┌──────────────────────────┐
│  t('decks.title')   [+]  │
│                          │
│  ┌────────────────────┐  │
│  │ [✅] Экзамен A2     │  │
│  │ 45 слов · 12 к повт │  │
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │ [⬜] Путешествия    │  │
│  │ 30 слов · —         │  │
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │ [✅] Бизнес         │  │
│  │ 20 слов · 5 к повт  │  │
│  └────────────────────┘  │
│                          │
│  ┌────────────────────┐  │
│  │ t('decks.import')   │  │
│  └────────────────────┘  │
│                          │
│  ┌─────┬─────┬────┬───┐ │
│  │ 🏠  │ 📦  │ 📊 │ ⚙ │ │
│  └─────┴─────┴────┴───┘ │
└──────────────────────────┘
```

### Список колод

- Отображается через `useLiveQuery` — реактивно обновляется при изменениях
- Каждая плитка показывает:
  - Checkbox `isActive` — включить/выключить колоду (toggle без захода внутрь)
  - Название колоды
  - Счётчик: слов в колоде + карточек к повторению (для активных колод)
- Тап по плитке → переход к `/decks/:deckId` (список слов внутри колоды)

### Действия

- **[+]** → создать новую колоду (inline-форма: name, description)
- **Редактировать** (кнопка на плитке) → изменить название/описание
- **Удалить** (кнопка на плитке) → с подтверждением, каскадно удаляет все слова и их CardState
- **Импорт CSV** → создаёт новую колоду или добавляет в существующую (dropdown выбора)

### Создание колоды

Inline-форма:

```
┌────────────────────────────┐
│ t('decks.name'):     [   ] │
│ t('decks.desc'):     [   ] │  ← опционально
│ [t('save')] [t('cancel')]  │
└────────────────────────────┘
```

## Deck Words (/decks/:deckId)

### Назначение
Просмотр и управление словами внутри конкретной колоды.

### Структура экрана

```
┌──────────────────────────┐
│  ← Экзамен A2       [+]  │  ← назад + кнопка добавить
│                          │
│  🔍 t('words.search')    │  ← поиск по тексту
│                          │
│  ┌────────────────────┐  │
│  │ olá — привет       │  │
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │ obrigado — спасибо │  │
│  └────────────────────┘  │
│  ...                     │
│                          │
│  ┌────────────────────┐  │
│  │ t('words.import')   │  │
│  └────────────────────┘  │
│                          │
│  ┌─────┬─────┬────┬───┐ │
│  │ 🏠  │ 📦  │ 📊 │ ⚙ │ │
│  └─────┴─────┴────┴───┘ │
└──────────────────────────┘
```

### Список слов

- Отображается через `useLiveQuery` с фильтром `db.words.where('deckId').equals(deckId)`
- Поиск: фильтрация по подстроке в `pt` или `translations[studyLanguage]` (клиентская)
- Каждая строка: PT — перевод на `studyLanguage`
- Действия на слове: редактировать, удалить, переместить в другую колоду

### Добавление/редактирование слова (WordForm)

Inline-форма:

```
┌─────────────────────────────┐
│ t('words.portuguese'): [  ] │
│ t('words.translation'): [ ] │  ← перевод на текущем studyLanguage
│ [t('save')] [t('cancel')]   │
└─────────────────────────────┘
```

- Валидация: оба поля (pt и перевод) обязательны
- Перевод сохраняется в `word.translations[studyLanguage]`
- `deckId` устанавливается автоматически из текущей колоды
- Проверка дубликатов по полю `pt`

### Перемещение слова

Действие "Переместить" на слове → dropdown с другими колодами → обновление `word.deckId`.

### Импорт CSV

- Кнопка `t('words.import')` открывает file input
- CSV-формат: `pt,translation` (два столбца, колода определяется текущим экраном)
- Перевод сохраняется в `translations[studyLanguage]`
- Первая строка может быть заголовком (автоопределение)
- Превью + подтверждение
- Валидные строки: `db.words.bulkAdd()` с `deckId` текущей колоды

### Данные

- **Сводка слов:**
  - Всего слов: `db.words.count()`
  - Изучено (Review): CardState с `state = Review`, уникальные по `wordId`
  - Новых: CardState с `state = New`, уникальные по `wordId`

- **Состояние карточек:** количество CardState в каждом FSRS-состоянии (New / Learning / Review / Relearning). Отображается горизонтальными CSS-барами.

- **Retention:** средний retention из `VocabularyMode.getStats()`

- **История сессий:** последние 30 сессий из таблицы `sessions`, отсортированные по `startedAt` (новые сверху). Для каждой: дата, результат (correct/total), процент.

Все данные через `useLiveQuery` для реактивности.

Все визуализации — чистый CSS (progress bars через width в %, без canvas/svg-библиотек).

```
│                          │
│  t('settings.sessionSize') │
│  [ 10 ]  t('cards')     │  ← числовой ввод

└──────────────────────────┘
```

Все надписи через `t()` из `useTranslation()`.

### Функционал (v1)

- **Язык интерфейса (uiLanguage):** dropdown с доступными языками (Русский, English). При смене — мгновенное переключение всех надписей через `i18next.changeLanguage(lang)`. Сохраняется в таблице `settings`. Без перезагрузки.

- **Тема:** light / dark / system. Переключение мгновенное (через `data-theme` на `<html>`). Сохраняется в `localStorage` (через хук `useTheme`) и в таблице `settings`.

- **Размер сессии:** число от 5 до 50. Сохраняется в таблице `settings`. Используется хуком `useSession` при старте сессии.

- **Сброс:** удаление всей БД (`db.delete()`) + перезагрузка страницы. Двойное подтверждение (confirm dialog).

### Функционал (future)

Следующие функции описаны для полноты архитектуры, но не реализуются в v1:

- **Язык переводов (studyLanguage):** dropdown для выбора языковой пары. В v1 зафиксирован как `'ru'`, не отображается в UI. Подробности — см. DATA_MODEL.md, секция "Смена studyLanguage (future)".

- **Экспорт данных:** выгрузка всех таблиц Dexie в JSON-файл. Скачивается через `<a download>` + `URL.createObjectURL(blob)`.

- **Импорт данных:** загрузка JSON-файла, валидация структуры, замена данных в БД. С предупреждением о перезаписи.
